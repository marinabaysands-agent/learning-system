# Marina's Personal Learning System — Design Doc

**日期：** 2026-04-28
**状态：** Draft v1
**目标：** 把输入的知识内化成自己的东西

---

## 一句话

一个以 Sands 为核心引擎的个人学习系统，通过「读→标注→复述→对撞→输出」的闭环，把被动阅读变成主动内化。

## 核心理念

- **学习 = 输出**（费曼学习法）— 不是读了就算学了，要能用自己的话说出来
- **知识要碰撞**（Zettelkasten）— 孤立的笔记没用，不同观点之间的连接才产生洞察
- **系统推着人走** — 不靠自律，靠机制。Sands 主动提醒、追问、安排计划

## 内化等级模型

| 等级 | 行为 | 说明 |
|------|------|------|
| L0 | 未读 | 内容在库里，还没看 |
| L1 | 读了/划线了 | 识别了有价值的点，但没有主动思考 |
| L2 | 用自己的话复述 | 费曼检验——能讲出来才算懂了 |
| L3 | 跨内容观点对撞 | 不同来源的观点产生碰撞和新认知 |
| L4 | 输出成文 | 写成文章/周记/分享，真正变成自己的 |

系统的目标：把每个内容从 L0 推向 L4。

## 系统架构

```
┌─────────────────────────────────────────────────┐
│              Marina's Learning System            │
├──────────┬──────────────────┬────────────────────┤
│  网站     │  Sands (飞书)     │  输出               │
│  读+标注  │  对话+提醒+计划   │  公开学习成果        │
├──────────┴──────────────────┴────────────────────┤
│              共享数据层（Cloudflare KV）           │
│  内容库 / 划线 / 复述 / 问答 / 阅读状态 / 学习计划  │
└─────────────────────────────────────────────────┘
```

### 三个界面

**1. 网站（learning.marinago.one）— 深度阅读场景**
- 阅读文章、划线高亮、添加批注
- 读后语音复述
- 回看笔记、浏览主题
- 查看学习计划和进度
- 展示公开学习成果

**2. Sands 飞书对话 — 随时随地场景**
- 「帮我存一下这篇文章」→ 抓取、整理、入库
- 「上周 Chesky 那篇讲砍项目是怎么说的？」→ 从笔记中检索
- 主动推送学习提醒和每日计划
- 费曼式对话：你讲我追问
- 观点对撞推送

**3. 输出展示 — 内化的证明**
- 复述、观点整理成公开内容
- 放在网站的「学习笔记」板块
- LinkedIn 周记素材来源

## 功能详述

### F1: 阅读状态追踪

**状态流转：**
```
未读 → 阅读中（记住滚动位置）→ 读完 → 已复述
```

**网站展示：**
- 内容列表页显示状态标签（彩色 badge）
- 「阅读中」的条目显示进度条
- 可按状态筛选

### F2: 划线高亮 + 批注

**交互：**
- 选中文字 → 弹出工具条 → 高亮（可选颜色）/ 添加批注
- 高亮持久化存储，刷新不丢失
- 可删除/编辑

**「我的笔记」汇总页：**
- 按文章分组 / 按时间线 / 按标签
- 每条笔记点击可跳转回原文位置

### F3: 读后语音复述

**交互：**
- 文章读完后，底部出现复述区域：

  > **用你自己的话说说：这篇你记住了什么？**
  >
  > 🎤 按住说话

- 录音 → Deepgram 转录（nova-2，中英混合）→ 显示文字
- 可重录 / 可编辑文字

**追问：**
- 复述完成后，展示 1-2 个预置的逼问问题
- 同样语音回答 → 转录保存

### F4: 每日学习计划

**生成逻辑（Sands cron，每天早上推送）：**
- 每天 25 分钟预算
- 混合安排：
  - 新内容阅读（约 15 分钟，1 篇）
  - 复习旧内容（约 10 分钟，复述或问答）
- 优先级：
  1. 读了一半的 → 继续读完
  2. 读完未复述的 → 做复述
  3. 未读的 → 按时间/兴趣推荐
  4. 间隔复习（距上次互动 3/7/14 天）

**推送渠道：** 飞书消息
**网站展示：** 首页「今日计划」卡片

### F5: 标签 + 主题关联

**标签系统：**
- 每篇内容预打 3-5 个标签
- 网站有 `/topics/` 页面，按标签聚合内容
- 标签举例：`leadership` `ai-product` `growth` `hiring` `decision-making`

**跨文章关联：**
- 相关文章推荐（基于共同标签）
- 观点对比卡片（同主题不同观点）

### F6: 分享功能

- 生成公开链接：你的划线 + 复述 + 笔记
- 别人只能看不能编辑
- 可选择分享整个笔记本 or 单篇笔记

- [ ] 读后语音复述（Deepgram 转录）

- [ ] 标签系统 + 主题页

## 技术方案

### 前端
- **框架：** 静态 HTML + vanilla JS（保持轻量，和现有站点一致）
- **划线：** Web Selection API + Range API
- **语音：** MediaRecorder API 录音 → 发送到 Worker → Deepgram 转录
- **状态：** Intersection Observer 追踪阅读进度
- **部署：** Cloudflare Pages（现有）

### 后端
- **Cloudflare Workers** — 处理 API 请求
  - `POST /api/highlight` — 保存划线
  - `POST /api/retell` — 上传语音 → Deepgram 转录 → 保存
  - `POST /api/qa` — 保存问答
  - `GET /api/notes` — 获取笔记数据
  - `PUT /api/progress` — 更新阅读进度
- **Cloudflare KV** — 存储所有用户数据
  - Key 设计：`content:{id}:highlights`, `content:{id}:retelling`, `user:plan`, etc.
- **Deepgram API** — 语音转文字（Worker 代理，不暴露 key）

### Sands 集成
- Sands 通过读取 KV 数据（或 Worker API）获取学习状态
- Cron 每天早上生成学习计划 → 写入 KV + 飞书推送
- Cron 检测 RSS 新 episode → 自动转录分析 → 入库

### 认证
- MVP 阶段：简单密码保护（你一个人用）
- 或者：不做认证，URL 不公开，数据按浏览器 fingerprint 区分

## 页面结构

```
learning.marinago.one/
├── /                    # 首页：今日计划 + 学习概览
├── /library/            # 内容库：所有文章，可按状态/标签筛选
├── /library/:id         # 单篇阅读页（划线 + 复述 + 问答）

├── /topics/             # 按标签聚合的主题页
└── /topics/:tag         # 单个主题下的所有内容
```
