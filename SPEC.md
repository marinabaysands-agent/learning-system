# Learning System — SPEC v1

## 这是什么

Marina 的个人学习系统。把播客/文章/书变成可消化的知识。

**核心循环：** 读 → 划线 → 批注 → 对话 → 内化

## 划线功能的问题分析

### 当前实现
1. 用户选中文字 → `selectionchange` 事件触发
2. 获取选中文本 + **计算纯文本偏移量**（start/end）
3. 保存 `{ text, position: { start, end }, color }`
4. 渲染时：遍历 container 的文本节点，按 start/end 用 `<mark>` 包裹

### Bug 根因
1. **跨节点高亮只标了第一个节点** — `applyMark` 遇到第一个匹配节点就 return，跨 `<em>`/`<strong>` 的选中只能标部分
2. **DOM 漂移** — 每次 render 先 remove 所有 mark 再重建，但如果文章本身有嵌套元素，normalize() 后的文本偏移量可能和保存时不一致
3. **多个高亮互相干扰** — 渲染第 N 个高亮后 DOM 变了，第 N+1 个的偏移量就不对了

### 解决方案选项

| 方案 | 复杂度 | 可靠性 | 说明 |
|------|--------|--------|------|
| A: 修复跨节点 + 排序渲染 | 中 | 中 | 修 applyMark 支持多节点，按 position.start 排序渲染 |
| B: 用 XPath + offset | 中 | 高 | 保存精确的 DOM 路径而非纯文本偏移，更抗 DOM 变化 |
| C: 用 CSS Custom Highlight API | 低 | 高 | 浏览器原生 API，无 DOM 修改，但兼容性限 Chrome 105+ |
| D: 用成熟库 (Rangy / mark.js) | 低 | 高 | 社区验证方案，不自己造轮子 |

**推荐：方案 A（先修跨节点 + 排序）作为短期修复，长期考虑 C 或 D。**

### 划线功能的分期
- **Phase 1（必须）：** 选中 → 标黄正确的文字 → 保存 → 刷新后恢复
- **Phase 2（重要）：** 批注、删除、颜色切换
- **Phase 3（可选）：** 学习对话（对高亮内容提问）

## 网页需要的页面

### 1. 首页（/）
- 欢迎语
- 继续阅读（最近未读完的 1-3 篇）
- 快速导航（内容库 / 笔记 / 主题）
- 学习热力图（最近 30 天的阅读/划线活动）

### 2. 内容库（/library/）
- 所有内容的卡片列表
- 筛选：类型（深度分析/书/视频/文章）+ 状态（未读/阅读中/已读）
- 卡片信息：标题、作者、标签、阅读进度
- 响应式：桌面 2 列，手机单列

### 3. 文章页（/articles/xxx.html）
- 文章正文阅读
- **划线高亮**（选中文字 → 工具栏 → 标黄/批注）
- 目录侧边栏（桌面）
- 阅读进度条
- 顶部进度保存

### 4. 笔记页（/notes/）
- 按文章分组显示所有高亮 + 批注
- 可跳转到原文位置

### 5. 主题页（/topics/）
- 标签云
- 点击标签 → 显示该标签下的文章

## 设计原则（见 DESIGN.md）

2. [ ] 划线功能：选 ABCDE → 标黄的就是 ABCDE，不是别的字
3. [ ] 划线刷新后能恢复
4. [ ] 首页有热力图
5. [ ] 全站视觉一致、干净
