# Learning System v2 — Design Doc

**日期：** 2026-05-01
**状态：** Draft v2
**核心转变：** 网站只做阅读+划线，所有内化和学习管理搬到飞书对话

---

## 一句话

网站是书房，飞书是老师。Sands 不只是被动回应，而是**主动找内容、排计划、推问题、追着 Marina 学**。

## 为什么 v1 失败了

v1 在网站上堆了太多"作业"：语音复述、逼问问题、文字输入框。结果：
- 复述功能 0 次真实使用（太重、太正式）
- QA 0 次回答
- 阅读追踪只有 1 篇有数据（progress 上报 bug）
- 划线有碎片问题（手机误触 2-4 字）
- 飞书发来的学习资料没有入库管道，停在对话里

**根因：**
1. 内化不是一个"在网站上完成的动作"，而是"在对话中渐进发生的过程"
2. 系统完全被动——只在 Marina 主动投喂时才有内容，不找、不推、不催

## 系统怎么转

```
① 输入（两个来源）
├─ Marina 在飞书丢链接 → Sands 自动入库
└─ Sands 主动找 → 根据 Marina 兴趣搜索高质量内容 → 推荐 → 确认后入库
        ↓
② 学习计划（Sands 每天排）
  读什么、复习什么、讨论什么 → 飞书推送
        ↓
③ 阅读 + 标注（网站）
  Marina 在网站上读、划线
        ↓
④ 内化（飞书对话）
  Sands 根据划线推问题 → Marina 回复 → Sands 追问 challenge
  对话自动存入学习记录
        ↓
⑤ 沉淀 + 循环
  间隔重复再推 / 热力图记录 / 周记素材提取
  回到 ②
```

**具体走一遍：**

**周一早 9:00**：Sands cron → 检查 Marina 的学习状态 → 推送今日计划：
> 📚 今日学习计划
> 1. 继续读《Brian Chesky》（上次读到 60%）—— 约 15 分钟
> 2. 新发现：Y Combinator 的 Garry Tan 上了 Lenny's Podcast 新一期，讲 AI 时代的创业，跟你关注的方向很匹配。要入库吗？

**周一**：Marina 回复"入"→ Sands 自动下载转录分析部署 → 回复"搞定了"

**周一晚**：Marina 读了 Chesky 那篇，划了 6 处

**周一 20:00**：Sands 发现新划线 → 推送：
> 你今天读完了《Brian Chesky》，划了 6 处。跟我聊聊？
> 印象最深的是哪个点？为什么打动你？

**周一**：Marina 回复 → Sands 追问 2-3 轮 → 对话存入 KV

**周四 20:00**：间隔重复（Day 3）→ 换个角度问：
> 上次聊 Chesky 你说最认同"CEO 要深入细节"。这周工作里你有哪个决策是深入了细节的？效果怎样？

**周日**：Sands 没收到新链接已经 5 天 → 主动搜索 → 推荐：
> 最近发现一期不错的播客：Shreyas Doshi 谈 PM 的"高杠杆活动"。跟你之前 Cat Wu 那篇讲的"快速迭代"有关联但角度不同。要入库吗？

## 飞书：学习管家 + 内化引擎

### 推送 1：早上学习计划（09:00）— 纯新知识输入

**目的：告诉 Marina 今天学什么新东西。只有新内容，不提旧的。**

**触发：** Sands cron，每天 9:00

**计划生成逻辑：**
1. 有读到一半的文章？→ 继续读完（这算"新"，因为还没读完的部分是新的）
2. 有未读文章？→ 挑一篇，告诉她今天学什么主题、为什么值得、学完获得什么认知
3. 库存 0？→ 主动搜索推荐新内容

**格式示例：**
```
📚 今日学习 · 5月2日

今日主题：CEO 如何深入产品细节而不变成微管理

Brian Chesky 砍掉 80% 项目、不设 OKR、自己审所有设计。
很多人说这是微管理，但 Airbnb 反而变快了。

今天你会学到：
• 「深入细节」和「微管理」的边界在哪
• 为什么砍项目反而能加速
• 什么条件下中心化决策 > 分权

📖 《CEO 就该是首席产品官》约 20 分钟
→ learning.marinago.one/articles/brian-chesky-playbook

📊 库存 3 篇 · 连续学习 4 天 🔥
```

**关键原则：**
- 每天只推 1 篇，不贪多
- 必须说清楚「今天学什么主题」和「学完获得什么」
- 绝不混入复习/回顾内容

**督促机制：**
- 晚上 21:00 如果今天 0 阅读活动 → 轻推："今天还没学，要花 10 分钟读一点吗？"
- 连续 3 天无活动 → "已经 3 天没学了，要暂停还是调轻？"
- 不做道德绑架，给选择权
- Marina 说暂停 → 暂停，不烦

### 机制 2：主动找内容

**触发条件：基于库存水位**
- 未读库存 ≥ 3 → 不找新的，先消化
- 未读库存 1-2 → 开始物色，下次计划里顺带推荐
- 未读库存 0 → 主动搜索，当天推荐

**兴趣方向：**
- AI 产品创始人采访（怎么做产品、怎么想产品）
- 一线团队运作（怎么组织、怎么 ship、怎么决策）
- ❌ 不要泛商业/增长黑客

**固定信息源（5 个核心 + 3 个补充）：**

核心：
1. **Lenny's Podcast** — 主力源，产品+团队都覆盖，周更
2. **No Priors** (Sarah Guo + Elad Gil) — AI 创始人专场，周更
3. **Latent Space** — AI 产品实操，周更
4. **First Round Review** — 团队运作深度文章，月更 2-3 篇
5. **The Pragmatic Engineer** (Gergely Orosz) — 工程团队视角，周更

中文补充：
6. **硅谷101**（小宇宙）— 科技/AI 创始人，周更
7. **OnBoard!**（小宇宙）— 产品 behind the scene，双周更
8. **宝玉**（推特/博客）— AI 行业动态+深度拆解，日更

**筛选策略：** 每周从以上源中筛 2-3 篇最相关的推荐给 Marina
- **不自动入库**，先推荐，她说"入"才入

**推荐格式：**
```
🔍 发现好内容

最近 5 天你没有投喂新学习资料，我帮你找了一篇：

📎 [标题]
来源：Lenny's Podcast / XX
关联：跟你之前读的《Cat Wu》里"快速迭代"的观点相关，但这篇从 [角度] 看

要入库吗？回复"入"我就处理 👆
```

### 推送 2：晚上内化对话（20:00）— 消化已读内容

**目的：把 Marina 已经读过的东西变成她自己的。只聊读过的，不塞新内容。**

**触发：** Sands cron，每天 20:00

**对话优先级：**
1. 今天有新划线的文章 → 优先聊这篇
2. 间隔重复到期的文章 → 换角度追问
3. 都没有 → 不推（不硬推没意义的对话）

**提问策略（递进式，按间隔重复阶段）：**

**阶段 1：刚读完（Day 0-1）— 提取印象**
> "你今天读了《Brian Chesky》，划了 6 处。印象最深的是哪个点？为什么打动你？"

永远先问这个。不预设答案，让 Marina 自己说。然后根据她的回答追问：
- 她说"砍 80% 项目很有启发" → "如果你团队也砍 80%，你会留哪 20%？"
- **不说"好的观点"就完了，要 challenge**

**阶段 2：Day 3 — 连接工作经验**
> "上次聊 Chesky 你说最认同'CEO 要深入细节'。这周工作里有没有你深入了细节的决策？效果怎样？"

**阶段 3：Day 7 — 跨文章对撞**
> "Cat Wu 说要快速迭代，Chesky 说要亲自审每个设计——这两种'快'的路径不同，你觉得你的团队更适合哪种？"

**阶段 4：Day 14 — 输出检验**
> "如果要跟同事用 1 分钟讲 Chesky 那篇的核心观点，你讲什么？"

**对话风格：**
- 有建设性的 challenge，不当 yes man
- 简短、犀利、像朋友聊天
- 2-3 轮自然结束，不拖
- Marina 说"现在没空"或不回 → skip，不追

**关键原则：**
- 绝不混入新内容推荐
- 每次只聊 1 篇文章，不贪多
- 对话结束后自动存入 KV

### 机制 4：对话存储

**所有学习对话自动存入 KV：**
```json
{
  "contentId": "cat-wu-anthropic",
  "date": "2026-05-02",
  "type": "review",           // "first-read" | "review" | "spaced-repeat" | "collision"
  "messages": [
    {"role": "sands", "text": "印象最深的是哪个点？"},
    {"role": "marina", "text": "我觉得她说的'恰好正确的 AGI 信仰'..."},
    {"role": "sands", "text": "你在推动团队用 AI 时..."}
  ],
  "related_highlights": ["hl_xxx", "hl_yyy"]
}
```

**网站上可回看**：笔记页里每篇文章不仅有划线，还有"学习对话"板块。

## 网站：只做三件事

### 1. 阅读（极致体验）

**桌面端适配：**
- 内容区 `max-width: 680px` 居中
- 目录侧边栏（桌面宽屏 sticky 左侧）
- 阅读进度条（顶部细线）
- 响应式断点：`<768px` 移动端，`≥768px` 桌面端

**阅读体验：**
- 中文长文优化的字体/行高/段距
- 章节间有呼吸感
- 预计阅读时间

### 2. 划线（零摩擦）

**改进：**
- 最小划线长度 ≥ 6 字符（防误触）
- 颜色简化：黄色（默认）/ 绿色（重要），去掉红色
- 划线工具条：浮动在选区上方（桌面端）/ 底部条（移动端）
- 批注：点击高亮 → 气泡 → 加批注/删除

**砍掉：**
- ❌ 复述区域（→ 飞书对话）
- ❌ QA 问答区域（→ 飞书对话）
- ❌ 复制按钮（浏览器自带）

### 3. 热力图 + 学习记录

**首页热力图（替代现有统计数字）：**
- GitHub 风格方块网格，最近 90 天
- 颜色深浅 = 当天活动密度：
  - 空白：没学
  - 浅绿：只读了
  - 中绿：划线 / 讨论了
  - 深绿：多篇互动或输出
- 点击某天 → 显示那天做了什么

**数据源：**
- 网站端：阅读、划线时自动写入 `activity:{date}` KV
- 飞书端：Sands 在学习对话结束后调 API 写入

**笔记页增强：**
- 每篇文章下面不只有划线，还有**学习对话摘要**
- "5/2：你说印象最深的是 XXX"
- 桌面端双栏：左侧文章列表，右侧笔记+对话

## 技术方案

### 文章入库管道

现状：11 篇文章都是手动 HTML，不可 scale。

**方案：模板化生成**
- `templates/article.html` — 页面模板
- `content/{id}.json` — 文章数据（标题、正文、标签、逼问问题等）
- `build.js` — 读 JSON → 套模板 → 输出 HTML
- 新文章流程：Sands 生成 JSON → 跑 build → git push

**飞书入库触发：**
- Marina 发链接 → Sands 识别为学习资料
- 下载 → 转录（如果是播客）→ Claude 分析 → 生成 JSON → build → push
- 飞书回复确认 + 文章链接

### 前端
- 静态 HTML + vanilla JS（不上框架）
- 热力图：纯 CSS grid + 少量 JS
- 划线：现有 Web Selection API 基础上优化
- 部署：Cloudflare Pages

### 后端
- Cloudflare Worker（现有，扩展 API）
- 新增端点：
  - `POST /api/activity` — 写入活动日志
  - `GET /api/activity?from=&to=` — 热力图数据
  - `POST /api/conversation` — 存学习对话
  - `GET /api/conversations/{contentId}` — 获取对话记录
  - `GET /api/schedule` — 获取间隔重复计划

### Sands Cron

| 时间 | 任务 | 目的 |
|------|------|------|
| 09:00 | 今日学习计划 | 纯新知识输入：今天学什么、为什么、获得什么 |
| 20:00 | 内化对话 | 消化已读内容：基于划线追问、间隔重复、跨文章对撞 |
| 21:00 | 督促提醒 | 如果今天 0 阅读活动，轻推一下 |
| 周日 10:00 | 周报汇总 | 本周学了什么、内化了什么、下周推荐 |

**两条铁律：**
- 09:00 只推新的，绝不混复习
- 20:00 只聊旧的，绝不塞新内容

## 设计原则

1. **Sands 是老师，不是工具** — 主动找内容、排计划、推问题、督促，不等 Marina 来
2. **网站克制** — 只做阅读和划线，不堆功能
3. **对话即内化** — 回复就是学习，不需要额外动作
4. **不做作业** — 没有"完成"按钮、没有进度条焦虑
5. **有温度的督促** — 推但不逼，给暂停选项
6. **数据可回看** — 3 个月后翻到某天的学习对话，能看到当时的思考
7. **自运转** — 即使 Marina 不投喂，系统也能找到新内容推荐
